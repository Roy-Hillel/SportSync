# Plan 4.1: Update Bootstrap Script for API-Football

## Objective

Fix `src/lib/bootstrap/seed-entities.ts` so it works correctly with the active `api-football` provider. Two concrete bugs to fix and one format verification:

1. `PRIORITY_COMPETITION_IDS` still contains SportRadar ID strings (e.g. `"sr:competition:7"`) — these must be replaced with API-Football integer string IDs.
2. `sleep(1100)` on line 69 (with a comment referencing SportRadar rate limits) must be removed — API-Football's 300 req/min limit does not require per-request delays in bootstrap.
3. `PRIORITY_COUNTRIES` must be verified against the country names API-Football actually returns in `item.country?.name` from `GET /leagues`.

**Requirements covered:** SEED-01, SEED-02, SEED-03, SEED-04

---

## Context

### What the script does today (no changes needed)

- Calls `getActiveProvider()` — already provider-agnostic (no change needed).
- Calls `provider.listCompetitions()` → API-Football `GET /leagues?current=true` → returns `ProviderEntity[]` with `providerId = String(item.league.id)` (integer strings like `"383"`, `"39"`).
- Calls `provider.listTeamsInCompetition(comp.providerId)` per priority competition → API-Football `GET /teams?league=<id>&season=<year>`.
- Upserts with conflict target `(providerId, provider)` — correct, matches the `subscribable_entities` schema.
- Sets `provider = provider.name = "api-football"` automatically via `upsertEntities(provider.name, ...)`.
- Sets `parentProviderId` on team rows to the competition's `providerId` — already correct per SEED-04.
- Gender/age suffix logic is provider-agnostic — keep as-is.

### What is broken

**Bug 1 — PRIORITY_COMPETITION_IDS has wrong IDs**

Current values (lines 131–137):
```ts
const PRIORITY_COMPETITION_IDS = new Set([
  "sr:competition:7",   // UEFA Champions League
  "sr:competition:18",  // UEFA Europa League
  "sr:competition:682", // UEFA Conference League
  "sr:competition:1",   // UEFA Euro
  "sr:competition:16",  // FIFA World Cup
  "sr:competition:34",  // CONMEBOL Copa América
]);
```

These are SportRadar format IDs. API-Football returns integer strings. The `topCompetitions` filter (`PRIORITY_COMPETITION_IDS.has(c.providerId)`) will never match any competition from API-Football, so _only_ the country-based filter applies. International competitions (Champions League, Europa League, etc.) use `country.name = "World"` in API-Football — which is not in `PRIORITY_COUNTRIES`. This means no teams from those competitions get seeded.

**Bug 2 — sleep(1100) with stale comment**

Lines 68–69:
```ts
      // Respect SportRadar trial rate limit (1 req/sec)
      await sleep(1100);
```

API-Football allows 300 requests/minute (5/sec). The bootstrap processes at most ~30–50 priority competitions. A 1.1s delay is unnecessary and would make bootstrap take ~55 seconds longer than needed with no benefit. Both the comment and the `await sleep(1100)` call must be removed.

**Issue 3 — PRIORITY_COUNTRIES country name format**

API-Football returns `country.name` from `item.country?.name` in the `/leagues` response. Known values:
- Domestic leagues: `"England"`, `"Spain"`, `"Germany"`, `"Italy"`, `"France"`, `"Netherlands"`, `"Portugal"`, `"Brazil"`, `"Argentina"`, `"USA"` — these match exactly.
- Israeli league: `"Israel"` — currently **not** in `PRIORITY_COUNTRIES`. Maccabi Haifa (team ID 4195) is in Ligat Ha'al (league ID 383). This league will not be seeded unless `"Israel"` is added.
- International club competitions (Champions League, Europa League, Conference League): `country.name = "World"` in API-Football — not in `PRIORITY_COUNTRIES`.
- International national team competitions (World Cup, Euros, Copa América): `country.name = "World"` as well.

The current `"International Clubs"` and `"International"` entries in `PRIORITY_COUNTRIES` are SportRadar-era values that will never match API-Football responses. They should be replaced with `"World"` (API-Football's actual value for all international competitions).
