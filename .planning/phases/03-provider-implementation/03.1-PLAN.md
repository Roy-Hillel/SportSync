# Plan 3.1: ApiFootballProvider Core

## Objective

Create the `ApiFootballProvider` class and supporting files (`client.ts`, `schemas.ts`, `index.ts`) in `src/lib/providers/api-football/`, fully implementing the `SportsDataProvider` interface. Also remove the per-second rate limit delay from the sync engine.

**Requirements covered:** PROV-01, PROV-02, PROV-03, PROV-04, PROV-05

## Context

**Interface contract** (`src/lib/providers/types.ts`):
- `SportsDataProvider` has 4 methods: `searchEntities`, `listCompetitions`, `listTeamsInCompetition`, `getSchedule`
- `ProviderEntity` fields: `providerId` (string), `entityType`, `displayName`, `logoUrl?`, `country?`, `parentProviderId?`
- `ProviderEvent` fields: `providerId` (string), `homeTeamName`, `awayTeamName`, `competitionName`, `homeTeamProviderId?`, `awayTeamProviderId?`, `competitionProviderId?`, `seasonProviderId?`, `startTime` (Date), `venue?`, `status` (EventStatus)
- `EventStatus`: `"scheduled" | "live" | "closed" | "cancelled" | "postponed" | "delayed"`
- All IDs are strings. API-Football returns integers -- always use `String(id)`.

**Reference implementation** (`src/lib/providers/sportradar/`):
- `client.ts` -- HTTP client with Zod validation, singleton factory, typed error class
- `api-types.ts` -- Zod schemas for all response shapes
- `index.ts` -- Provider class, status mapper, event mapper, singleton factory

**API-Football specifics:**
- Base URL: `https://v3.football.api-sports.io`
- Auth header: `x-apisports-key` (NOT Bearer, NOT `x-api-key`)
- Env var: `API_FOOTBALL_KEY` (already exists in `.env.local`)
- Response wrapper: `{ response: [...] }` for all endpoints
- Nullable fields confirmed from real API calls: `fixture.venue.id` (null), `league.flag` (null), `goals.home/away` (null pre-match), `fixture.venue.name/city` (may be absent)
- Season param: plain integer year (`?season=2025`), no season ID resolution needed

**Rate limit change:**
- `src/lib/sync/engine.ts` line 73: `if (i > 0) await new Promise((r) => setTimeout(r, 1100));`
- This 1.1s per-entity delay was for SportRadar's 1 req/sec limit
- API-Football allows 300 req/min -- the cron cadence (5h, typically 4-8 entities) never approaches this
- Remove the delay line and its comment block (lines 70-73)

## Tasks

### Task 1: Create `src/lib/providers/api-football/client.ts`
**File:** `src/lib/providers/api-football/client.ts`
**Action:** Create

Create an HTTP client mirroring `src/lib/providers/sportradar/client.ts` with these differences:

```typescript
import { ZodSchema } from "zod";

const BASE_URL = "https://v3.football.api-sports.io";
```

**`ApiFootballApiError` class:**
- Extends `Error`
- Constructor takes `message: string`, `public readonly statusCode: number`, `public readonly endpoint: string`
- Sets `this.name = "ApiFootballApiError"`

**`ApiFootballClient` class:**
- Constructor takes `apiKey: string`, throws if empty
- `async get<T>(endpoint: string, params: Record<string, string | number>, schema: ZodSchema<T>): Promise<T>` method:
  - Build URL from `BASE_URL + endpoint`
  - Set query params from `params` object using `url.searchParams.set(k, String(v))`
  - Fetch with header `{ "x-apisports-key": this.apiKey }` and `next: { revalidate: 300 }`
  - On non-ok: throw `ApiFootballApiError`
  - Parse response JSON through `schema.safeParse(raw)` -- throw descriptive error on failure
  - Return `parsed.data`

Note: The `get` method signature differs from Sportradar's -- it takes a `params` object as second arg (before schema) instead of embedding params in the endpoint string. This is cleaner for API-Football's query param pattern.

**Singleton factory:**
```typescript
let _client: ApiFootballClient | null = null;

export function getApiFootballClient(): ApiFootballClient {
  if (!_client) {
    const key = process.env.API_FOOTBALL_KEY;
    if (!key) throw new Error("API_FOOTBALL_KEY environment variable is not set");
    _client = new ApiFootballClient(key);
  }
  return _client;
}
```

Export: `ApiFootballApiError`, `ApiFootballClient`, `getApiFootballClient`.

### Task 2: Create `src/lib/providers/api-football/schemas.ts`
**File:** `src/lib/providers/api-football/schemas.ts`
**Action:** Create

Define Zod schemas for three API-Football response shapes. Import `z` from `"zod"`.

**Fixture response (`GET /fixtures`):**

Sub-schemas (not exported individually):
- `FixtureStatusSchema`: `{ short: z.string(), long: z.string().optional(), elapsed: z.number().nullable().optional() }`
- `FixtureVenueSchema`: `{ id: z.number().nullable(), name: z.string().nullable().optional(), city: z.string().nullable().optional() }`
  - CRITICAL: `id` is `z.number().nullable()` -- confirmed null on some Champions League fixtures (PROV-03)
- `FixtureLeagueSchema`: `{ id: z.number(), name: z.string(), country: z.string().optional(), logo: z.string().optional(), flag: z.string().nullable().optional(), season: z.number(), round: z.string().optional() }`
  - CRITICAL: `flag` is `z.string().nullable().optional()` -- null for international competitions (PROV-03)
- `FixtureTeamSchema`: `{ id: z.number(), name: z.string(), logo: z.string().optional(), winner: z.boolean().nullable().optional() }`
- `FixtureGoalsSchema`: `{ home: z.number().nullable(), away: z.number().nullable() }` -- both null pre-match

Composite:
- `FixtureItemSchema`: `{ fixture: { id: z.number(), date: z.string(), timestamp: z.number().optional(), timezone: z.string().optional(), venue: FixtureVenueSchema, status: FixtureStatusSchema }, league: FixtureLeagueSchema, teams: { home: FixtureTeamSchema, away: FixtureTeamSchema }, goals: FixtureGoalsSchema }`

Exports:
- `export type ApiFootballFixtureItem = z.infer<typeof FixtureItemSchema>;`
- `export const FixturesResponseSchema = z.object({ response: z.array(FixtureItemSchema) });`

**Leagues response (`GET /leagues`):**
- `LeagueSchema`: `{ league: { id: z.number(), name: z.string(), type: z.string().optional(), logo: z.string().optional() }, country: z.object({ name: z.string().optional(), code: z.string().nullable().optional(), flag: z.string().nullable().optional() }).optional() }`
- `export const LeaguesResponseSchema = z.object({ response: z.array(LeagueSchema) });`

**Teams response (`GET /teams?league&season`):**
- `TeamResponseItemSchema`: `{ team: { id: z.number(), name: z.string(), logo: z.string().optional(), country: z.string().nullable().optional() } }`
- `export const TeamsResponseSchema = z.object({ response: z.array(TeamResponseItemSchema) });`

### Task 3: Create `src/lib/providers/api-football/index.ts`
**File:** `src/lib/providers/api-football/index.ts`
**Action:** Create

Implement the `ApiFootballProvider` class. Mirror `src/lib/providers/sportradar/index.ts` structure.

**Imports:**
```typescript
import type { SportsDataProvider, ProviderEntity, ProviderEvent, EventStatus } from "../types";
import { getApiFootballClient } from "./client";
import { FixturesResponseSchema, LeaguesResponseSchema, TeamsResponseSchema, type ApiFootballFixtureItem } from "./schemas";
```

**`mapStatus(short: string): EventStatus` function:**
- `"NS"` -> `"scheduled"`
- `"1H"`, `"HT"`, `"2H"`, `"ET"`, `"P"` -> `"live"`
- `"FT"`, `"AET"`, `"PEN"` -> `"closed"`
- `"PST"` -> `"postponed"`
- `"CANC"` -> `"cancelled"`
- `"ABD"`, `"SUSP"` -> `"delayed"`
- default -> `"scheduled"`

**`mapFixture(item: ApiFootballFixtureItem): ProviderEvent | null` function:**
- Parse `item.fixture.date` as `new Date()` -- return null if invalid
- Return `ProviderEvent` with all IDs stringified via `String()`:
  - `providerId: String(item.fixture.id)`
  - `homeTeamName: item.teams.home.name`
  - `awayTeamName: item.teams.away.name`
  - `competitionName: item.league.name`
  - `homeTeamProviderId: String(item.teams.home.id)`
  - `awayTeamProviderId: String(item.teams.away.id)`
  - `competitionProviderId: String(item.league.id)`
  - `seasonProviderId: String(item.league.season)`
  - `startTime` (the parsed Date)
  - `venue: item.fixture.venue.name ?? undefined` (convert null to undefined)
  - `status: mapStatus(item.fixture.status.short)`

**`ApiFootballProvider` class implementing `SportsDataProvider`:**
- `readonly name = "api-football";`
- `private get client() { return getApiFootballClient(); }`

- `searchEntities(query: string)`: Filter from `listCompetitions()` by lowercase includes -- same pattern as Sportradar (interface completeness only; app uses DB cache for real search)

- `listCompetitions()`: Call `this.client.get("/leagues", { current: "true" }, LeaguesResponseSchema)`. Map response items to `ProviderEntity` with `entityType: "competition"`, `country: item.country?.name`.

- `listTeamsInCompetition(competitionProviderId: string)`: Use `season = new Date().getFullYear()` (PROV-04 -- no season ID resolution). Call `this.client.get("/teams", { league: competitionProviderId, season }, TeamsResponseSchema)`. Map to `ProviderEntity` with `entityType: "team"`, `parentProviderId: competitionProviderId`, `logoUrl: item.team.logo`, `country: item.team.country ?? undefined`.

- `getSchedule(entity: ProviderEntity, from: Date, to: Date)`: 
  - Derive `season = from.getFullYear()`
  - Format dates as `YYYY-MM-DD` strings using `d.toISOString().split("T")[0]`
  - Build params: if `entity.entityType === "competition"` use `{ league: entity.providerId, season, from: fmtFrom, to: fmtTo }`, else use `{ team: entity.providerId, season, from: fmtFrom, to: fmtTo }`
  - Call `this.client.get("/fixtures", params, FixturesResponseSchema)`
  - Map through `mapFixture`, filter out nulls

**Singleton factory:**
```typescript
let _provider: ApiFootballProvider | null = null;

export function getApiFootballProvider(): ApiFootballProvider {
  if (!_provider) _provider = new ApiFootballProvider();
  return _provider;
}
```

Export: `ApiFootballProvider`, `getApiFootballProvider`.
